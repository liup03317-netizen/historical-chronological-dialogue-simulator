# 历史的回音 — 编年体跨时代对话模拟器

## 概述

本技能实现一个横跨千年的学术讨论沙龙：用户提出一个问题，系统通过 LLM 流式生成 5-8 位代表不同历史时期的关键人物，以编年体递进顺序依次回答，形成从古代到现代的认知演进脉络。每位人物采用第一人称、口语化对话，保留其历史口癖与人格特质。

核心技术架构融合四个维度：
1. **LLM 异步流式生成** — Gemini SDK 流式输出，自定义分隔符 `|||` 划分人物块
2. **正则表达式渐进解析** — 非阻塞前瞻断言切割，实时提取 Name/WikiTitle/Era/Text
3. **百科全书动态媒体检索** — 维基百科→百度百科→快懂百科→搜狗百科四级降级链路
4. **拟真高阶打字机渲染** — 标点自适应停顿 + 正态延时抖动

## 工作流程

### 第一步：接收用户提问，构建 Prompt

当用户提出一个适合跨时代讨论的知识性问题（如"原子是什么？"、"地球是怎么形成的？"、"人类对光的认知经历了怎样的变迁？"），按以下步骤执行：

1. 将用户问题注入 Prompt 模板（详见 `references/prompt-design.md`）
2. 使用 `@google/genai` SDK 调用 Gemini 模型，启用流式传输 (`generateContentStream`)
3. 在 Prompt 中强制约束输出格式：`|||` 分隔，每个块含 Name / WikiTitle / Era / Text 四个键值对

### 第二步：渐进式流解析

流式传输过程中，原始文本以随机字节块不断累加至 Buffer。使用非阻塞正则表达式对 Buffer 进行渐进式解析，实现边接收边渲染：

1. 兼容模型偶尔出现的分隔符变形：`buffer.replace(/\|\|\|/g, '\n').replace(/---/g, '\n')`
2. 利用前瞻断言切割：`buffer.split(/(?=Name:\s*)/i)`
3. 对每个切割块提取 Name / WikiTitle / Era / Text
4. 根据是否为最后一个块或流是否结束，标记 `isComplete`
5. 容错降级：若模型未正确输出 `Text:` 前缀，自动过滤 Name/Era/WikiTitle 行后拼接剩余文本

**完整解析算法** 详见 `references/progressive-parser.md`。

### 第三步：百科全书媒体检索

对每个解析出 `WikiTitle` 的人物，按以下优先级依次尝试百科数据源，任一成功即返回：

**降级链路**：维基百科 → 百度百科 → 快懂百科 → 搜狗百科

1. **维基百科（首选）**：英文 WikiTitle + `redirects=1` 自动跨语种重定向至中文页面，提取 150-200px 肖像和 2 句式中文摘要
2. **百度百科（降级1）**：使用中文 Name 字段查询 `BaikeLemmaCardApi`，提取 `image` 和 `abstract`（不支持 CORS，纯前端需代理）
3. **快懂百科（降级2）**：通过 HTML 页面抓取 `<meta og:image>` 和 `<meta description>`（无公开 JSON API，需 WebFetch 工具）
4. **搜狗百科（降级3）**：使用 `openapi/lemma` API 或 HTML 抓取

**完整集成方案** 详见 `references/wikipedia-integration.md`。

### 第四步：生成完整 HTML 应用

将所有组件整合为一个自包含的 HTML 文件，使用 `assets/historical-dialogue-template.html` 作为基础模板：

1. **对话气泡布局**：历史人物气泡左对齐（含头像+名称+时代标签+内容），用户气泡右对齐
2. **头像渲染**：优先使用维基百科肖像，fallback 使用首字取母圆环
3. **悬浮卡片 (Tooltip Popover)**：桌面端 Hover 触发，绝对定位在侧边，含旋转45度三角形指示标
4. **打字机动画**：逐字渲染，普通字符 85-130ms 随机延时（正常人跟读速度），句末标点 500-900ms 停顿，句中停顿 280-450ms

**打字机算法** 详见 `references/typewriter-algorithm.md`。

### 第五步：预览与交付

1. 将生成的 HTML 文件写入项目目录
2. 使用 `preview_url` 工具展示最终效果
3. 使用 `deliver_attachments` 工具交付文件

## 技术栈

| 层级 | 技术 | 用途 |
|------|------|------|
| LLM | `@google/genai` SDK | 流式对话生成 |
| 解析 | 正则表达式前瞻断言 | 非阻塞渐进解析 |
| 媒体 | 百科全书四级降级链（维基→百度百科→快懂百科→搜狗百科） | 肖像 + 简介悬浮卡片 |
| 样式 | Tailwind CSS (CDN) | 响应式布局与学术色调 |
| 动画 | React `useState` + `useEffect` | 拟真打字机渲染 |

## 视觉设计规范

- **主色调**：学术暗调 `#2A2A22` 背景 + `#f5f5f0` 前景文字
- **气泡底色**：`#3D3D2E`（历史人物）、`#E5E5D8`（用户端）
- **悬浮卡片**：`#5A5A40` 底色 + `#f5f5f0` 文字
- **时代标签**：`#A89060` 文字色
- **头像**：48px 圆形，`object-cover`，维基百科肖像或首字取母
- **字体**：中文优先使用衬线体 `serif`，体现学术书卷气

## 环境要求

- `GEMINI_API_KEY` 环境变量必须已配置（用于 `@google/genai` SDK）
- 浏览器需支持 `fetch` 和 ES Modules
- 若无 API Key，降级为纯前端模拟（使用预设历史人物数据）

## 注意事项

- WikiTitle 必须使用标准英文维基百科页面标题，以便中文维基正确重定向
- 若维基百科不可达，自动降级至百度百科→快懂百科→搜狗百科
- 百度百科/快懂百科/搜狗百科使用中文 Name 字段查询，而非英文 WikiTitle
- 流式解析必须处理不完整数据块，不能等全部接收完毕再渲染
- 维基百科请求需设置 `origin=*` 以支持跨域；百度百科等不支持 CORS，纯前端需代理
- 打字机动画需在组件卸载时清理 `setTimeout`
- 所有输出（Name / Era / Text）强制简体中文
- 用户端头像使用精选人文感少年肖像缩略图
