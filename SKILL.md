# 历史的回音 — 编年体跨时代对话模拟器

## 概述

本技能实现一个横跨千年的学术讨论沙龙：用户提出一个问题，系统通过 LLM 生成 5-8 位代表不同历史时期的关键人物，以编年体递进顺序依次回答，形成从古代到现代的认知演进脉络。每位人物采用第一人称、口语化对话，保留其历史口癖与人格特质。

核心技术架构融合四个维度：
1. **LLM 对话生成** — 由执行本技能的 agent 自身生成对话内容，无需外部 API
2. **正则表达式渐进解析** — 非阻塞前瞻断言切割，实时提取 Name/WikiTitle/Era/Text
3. **百科全书动态媒体检索** — 维基百科→百度百科→快懂百科→搜狗百科四级降级链路
4. **拟真高阶打字机渲染** — 标点自适应停顿 + 正态延时抖动

## ⚠️ 重要：Agent 驱动模式

**本技能不需要任何外部 LLM API（如 Gemini、OpenAI 等）。** 执行本技能的 agent 本身就是 LLM，应直接生成对话内容并注入 HTML 模板。

**禁止**：
- ❌ 要求用户提供 API Key
- ❌ 在 HTML 中调用外部 LLM API
- ❌ 因缺少 API Key 而拒绝执行

**正确做法**：
- ✅ Agent 使用自身能力生成 5-8 位历史人物的对话内容
- ✅ 将生成的内容格式化为 JS 数组，注入 HTML 模板的 `FIGURES_DATA` 变量
- ✅ 将用户问题注入 HTML 模板的 `USER_QUESTION` 变量

## 工作流程

### 第一步：生成对话内容

当用户提出一个适合跨时代讨论的知识性问题（如"原子是什么？"、"地球是怎么形成的？"、"人类对光的认知经历了怎样的变迁？"），agent 自身生成 5-8 位历史人物的对话：

1. **选择人物**：按编年体顺序，选择 5-8 位代表关键历史转折点的先贤
2. **格式约束**：每个人物包含 Name（中文名）、WikiTitle（英文维基百科标题）、Era（时代）、Text（第一人称对话，3-6 句，口语化，简体中文）
3. **内容要求**：
   - 每位人物保留其历史口癖和人格特质
   - 人物之间有互动（点头、反驳、补充前人观点）
   - 从最早期到现代，形成认知演进脉络
   - 所有文本强制简体中文

**Prompt 模板** 详见 `references/prompt-design.md`。

### 第二步：注入 HTML 模板

1. 读取 `assets/historical-dialogue-template.html` 模板
2. 将生成的人物数据格式化为 JS 对象数组，替换模板中的 `{{FIGURES_PLACEHOLDER}}` 占位符
3. 将用户问题替换模板中的 `{{QUESTION_PLACEHOLDER}}` 占位符
4. 数据格式示例：

```javascript
{
  name: '德谟克利特',
  wikiTitle: 'Democritus',
  era: '约公元前460年',
  text: '朋友，在我看来...',
  isComplete: true
}
```

### 第三步：百科全书媒体检索

HTML 模板内置了百科四级降级链路，**运行时自动执行**，agent 无需手动处理：

**降级链路**：维基百科 → 百度百科 → 快懂百科 → 搜狗百科

1. **维基百科（首选）**：英文 WikiTitle + `redirects=1` 自动跨语种重定向至中文页面
2. **百度百科（降级1）**：中文 Name 字段查询 `BaikeLemmaCardApi`
3. **搜狗百科（降级2）**：`openapi/lemma` API 查询
4. **全部失败**：使用首字取母头像，不显示悬浮卡片

**完整集成方案** 详见 `references/wikipedia-integration.md`。

### 第四步：预览与交付

1. 将生成的 HTML 文件写入项目目录
2. 使用 `preview_url` 工具展示最终效果
3. 使用 `deliver_attachments` 工具交付文件

## 技术栈

| 层级 | 技术 | 用途 |
|------|------|------|
| 内容 | Agent 自身 LLM 能力 | 对话内容生成（无需外部 API） |
| 解析 | 正则表达式前瞻断言 | 非阻塞渐进解析 |
| 媒体 | 百科全书四级降级链（维基→百度百科→搜狗百科） | 肖像 + 简介悬浮卡片 |
| 样式 | 内联 CSS（零CDN依赖） | 响应式布局与学术色调 |
| 动画 | 原生 JS `setTimeout` | 拟真打字机渲染 |

## 模板特性

- **零CDN依赖**：不依赖 React、Babel、Tailwind 或任何外部 JS/CSS 库
- **零API依赖**：不需要 Gemini/OpenAI 等 API Key
- **完全离线可用**：核心功能（打字机动画、对话展示、总结卡片）无需网络
- **联网增强**：有网络时自动获取百科肖像和简介，无网络时降级为首字取母头像
- **链式驱动**：前一位人物打字完成后，600ms 仪式感延迟，下一位入场

## 视觉设计规范

- **主色调**：学术暗调 `#2A2A22` 背景 + `#f5f5f0` 前景文字
- **气泡底色**：`#3D3D2E`（历史人物）、`#E5E5D8`（用户端）
- **悬浮卡片**：`#5A5A40` 底色 + `#f5f5f0` 文字
- **时代标签**：`#A89060` 文字色
- **头像**：48px 圆形，`object-cover`，百科肖像或首字取母
- **字体**：中文优先使用系统衬线体，体现学术书卷气

## 打字机参数

| 字符类型 | 示例 | 延时范围 | 设计意图 |
|----------|------|----------|----------|
| 普通字符 | 汉字、字母 | 85-130ms | 正常人跟读速度，约7-10字/秒 |
| 句末标点 | 。！？.!? | 500-900ms | 模拟人类视线停留与呼吸节奏 |
| 句中停顿 | ，；：—-,;:- | 280-450ms | 模拟语段间歇与思考微停 |

## 注意事项

- **绝不要**因缺少 API Key 而拒绝执行或降级——agent 自身即可生成内容
- WikiTitle 必须使用标准英文维基百科页面标题，以便中文维基正确重定向
- 若维基百科不可达，HTML 运行时自动降级至百度百科→搜狗百科
- 所有输出（Name / Era / Text）强制简体中文
- 打字机动画在页面卸载时自动清理定时器
- 模板中的 `{{FIGURES_PLACEHOLDER}}` 和 `{{QUESTION_PLACEHOLDER}}` 必须被替换为实际数据
